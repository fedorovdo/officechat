# Groups Foundation OfficeChat

Groups Foundation v0.1 добавляет основу групп и участников без сообщений, WebSocket и файлов.

## Модель групп

Группа содержит:

- `name` - отображаемое название;
- `slug` - уникальный неизменяемый идентификатор;
- `description` - необязательное описание;
- `is_private` - приватная или публичная группа;
- `is_active` - активность группы;
- `created_by_user_id` - создатель группы.

`slug` нельзя изменить после создания.

## Глобальные роли и роли в группе

Глобальные роли пользователя:

- `superadmin`
- `admin`
- `group_owner`
- `moderator`
- `user`
- `bot`

Роли участника внутри группы:

- `owner`
- `moderator`
- `member`

Глобальная роль отвечает за административные права во всей системе. Роль в группе отвечает за права внутри конкретной группы.

## Права доступа

`superadmin` и `admin`:

- видят все группы;
- создают группы;
- редактируют все группы;
- управляют всеми участниками групп.

`owner` группы:

- редактирует базовую информацию своей группы;
- добавляет и удаляет участников;
- меняет роли участников внутри группы;
- не может удалить последнего владельца группы.

`moderator` группы:

- видит участников своей группы;
- пока не управляет участниками.

Обычный участник:

- видит группы, где он состоит;
- видит участников этих групп.

`bot`:

- может быть добавлен в группу как участник;
- специального поведения пока нет.

## API endpoints

Группы:

- `GET /api/groups`
- `POST /api/groups`
- `GET /api/groups/{group_id}`
- `PATCH /api/groups/{group_id}`

Участники:

- `GET /api/groups/{group_id}/members`
- `POST /api/groups/{group_id}/members`
- `PATCH /api/groups/{group_id}/members/{member_id}`
- `DELETE /api/groups/{group_id}/members/{member_id}`

## Ограничения v0.1

- Нет сообщений.
- Нет WebSocket.
- Нет файлов и вложений.
- Нет поиска пользователей для групп, UI использует простой ввод username.
- Нет удаления групп.
