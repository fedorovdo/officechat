# Лицензионная политика OfficeChat

OfficeChat распространяется под Apache License 2.0.

## Предпочтительные лицензии зависимостей

В core-проекте предпочтительны зависимости с permissive лицензиями:

- MIT
- Apache-2.0
- BSD
- PostgreSQL License
- ISC

## Чего избегать в core

В core-проекте следует избегать:

- AGPL;
- SSPL;
- source-available лицензий;
- лицензий с неясными коммерческими ограничениями;
- зависимостей, которые требуют публикации всего производного продукта при обычном self-hosted использовании.

## Проверка новых зависимостей

Перед добавлением зависимости нужно проверить:

- лицензию пакета;
- активность сопровождения;
- наличие известных security advisories;
- возможность offline/self-hosted работы;
- отсутствие обязательной зависимости от внешнего SaaS.

Спорные зависимости должны обсуждаться до добавления в core.
