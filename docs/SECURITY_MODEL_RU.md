# Модель безопасности OfficeChat

OfficeChat ориентирован на self-hosted и локальные корпоративные развертывания. Безопасность должна строиться слоями, даже если первый MVP работает только в LAN.

## Базовые решения

- Саморегистрация отключена по умолчанию.
- Пользователей создает администратор.
- Секреты задаются через переменные окружения.
- Файлы MVP хранятся в локальном Docker volume.
- Production-доступ из интернета должен проходить через reverse proxy с TLS.

## Будущие зоны безопасности

- Хеширование паролей современным алгоритмом.
- Роли и права доступа.
- Защита сессий и refresh token flow.
- Rate limiting для auth endpoint.
- Audit log для администрирования.
- Настройки CORS под конкретный deployment.
- Backup/restore и контроль доступа к backup.

## LDAP/AD

LDAP/AD не входит в MVP, но auth-архитектура должна позволять добавить внешний provider позже. Локальная аутентификация и внешний provider не должны быть жестко связаны с бизнес-логикой чатов.

## Интернет-развертывание

Перед публикацией в интернет нужны:

- TLS;
- secure headers;
- корректные trusted proxy настройки;
- закрытие прямого доступа к внутренним сервисам;
- надежная политика паролей;
- обновления контейнеров и зависимостей.
