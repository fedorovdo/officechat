# Закреплённые сообщения OfficeChat

Pinned Messages v0.1 добавляет закрепление сообщений в группах, личных чатах и обсуждениях. Это быстрый способ оставить важную инструкцию, ссылку или решение видимым в верхней части выбранного чата без создания отдельного канала или темы.

## Модель данных

Закрепления хранятся в таблице `pinned_messages`.

Основные поля:

- `chat_type` - `group`, `direct` или `discussion`;
- `chat_id` - идентификатор конкретного чата;
- `message_id` - идентификатор закреплённого сообщения;
- `pinned_by_user_id`, `pinned_by_username`, `pinned_by_display_name` - снимок автора закрепления;
- `pinned_at` - время закрепления;
- `note` - необязательная короткая заметка до 300 символов.

В одном чате одно сообщение может быть закреплено только один раз. Лимит количества закреплений задаётся переменной `PINNED_MESSAGES_MAX_PER_CHAT`, по умолчанию `20`.

## Права

Закреплять, откреплять и менять заметку может только активный не-bot пользователь с эффективным правом `can_pin_messages` и обычным доступом к выбранному чату.

Важно:

- роль `admin` сама по себе не даёт право закрепления;
- `superadmin` получает `can_pin_messages` неявно, но direct/discussion приватность не обходится административной ролью;
- bot users не могут закреплять и откреплять сообщения;
- сообщения от bot users можно закреплять, если это делает человек с правом `can_pin_messages`;
- удалённые и архивные сообщения не закрепляются;
- при удалении или архивации сообщения связанные закрепления автоматически снимаются.

## API

```text
GET    /api/pins?chat_type=group&chat_id=<uuid>
POST   /api/pins
PATCH  /api/pins/{pin_id}
DELETE /api/pins/{pin_id}
```

Создание:

```json
{
  "chat_type": "group",
  "chat_id": "...",
  "message_id": "...",
  "note": "Важно для дежурной смены"
}
```

Ответ содержит превью сообщения, автора закрепления, время, заметку и публичные сведения об авторе сообщения. Тело архивного сообщения в превью не раскрывается.

## WebSocket

Комнатные каналы групп, личных чатов и обсуждений получают события:

```json
{ "type": "message.pinned", "chat_type": "group", "chat_id": "...", "message_id": "...", "pin": {} }
```

```json
{ "type": "message.pin_updated", "chat_type": "group", "chat_id": "...", "message_id": "...", "pin": {} }
```

```json
{ "type": "message.unpinned", "chat_type": "group", "chat_id": "...", "message_id": "...", "pin_id": "..." }
```

Персональный канал `/api/ws/me` в v0.1 не рассылает отдельные события закрепления; открытый чат обновляет pinned strip через свой комнатный WebSocket.

## Audit Log

Действия записываются в `audit_events`:

- `message.pinned`;
- `message.unpinned`;
- `message.pin_note_updated`.

В details сохраняются только безопасные поля: тип чата, `chat_id`, `message_id`, `pin_id`, длина заметки до/после. Тело сообщения и полный текст заметки в аудит не пишутся.

## UI

В `/ru/app` над списком сообщений отображается компактная полоска закреплённых сообщений. Она показывает последнее закрепление, количество дополнительных закреплений и открывает список всех закреплений чата.

Пользователь с `can_pin_messages` видит действия:

- закрепить сообщение;
- открепить сообщение;
- добавить или изменить короткую заметку.

Кнопка перехода использует существующий механизм jump-to-message и подсвечивает целевое сообщение. В сообщении отображается небольшой признак `Закреплено`.

## Ограничения v0.1

- нет отдельного уведомления о закреплении через `/api/ws/me`;
- нет поиска только по закреплённым сообщениям;
- нет отдельной страницы администрирования закреплений;
- нет массового переноса закреплений между чатами;
- multi-instance доставка WebSocket-событий позже должна использовать Valkey pub/sub.

## Центр уведомлений

Закрепление сообщения может создать notification типа `pin`, если у получателя включена настройка `pins_enabled`. По умолчанию она выключена. Такое уведомление не меняет chat unread counters.
